Decode and validate hashed user data (PII) from requests to Google Ads, GA4, Meta or other marketing platforms.

  • Paste a Google em=… value from Tag Assistant or the Network tab (with or without the em= prefix). Format: tv.1~em.HASH~pn.HASH~…
  • Tokens: em email, pn phone, fn0/ln0 first/last name, sa0 street, ct0/rg0/pc0/co0 city/region/postal code/country.
  • Or click a captured request below — Google requests load their em value, Meta and other detector requests load their advanced-matching fields, each validated with that provider's own hashing rules.
  • The HEX/B64URL pill marks the hash encoding — auto-detection sends Base64URL, manually provided sha256_* values are Hex.
  • Fill the optional "verify fields" below with plaintext — values are hashed and matched against each token (MATCH/ERR).

    Inspect a user_data object from dataLayer.push, gtag(), or raw JSON.

    Show more
    • Paste a dataLayer.push({…}) call, a gtag('event', '…', {…}) call, or just the {…} object. JS-style syntax (unquoted keys, single quotes) is normalized automatically. You can also just paste the user-rovided-data variable content from Google Tag Assistant.
    • Address fields (first_name, last_name, country, postal_code, street, city, region incl. their sha256_* variants) must sit inside user_data.address — flat placement triggers a structure error.
    • Compliance pill: green = Email or full address present · grey = only Postal Code missing (still processed) · yellow = warning, fields likely insufficient for matching.
    • The HEX/B64URL pill marks each sha256_* hash encoding — manually pre-hashed values are usually Hex.
